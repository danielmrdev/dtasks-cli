  <files></files>
  <action>
From main (already checked out after merge), run the release target:

```bash

```

1. `git tag v0.3.0`
2. `git push origin v0.3.0`

The tag push triggers `release.yml` on GitHub Actions, which:
1. Matrix-builds 6 binaries (darwin/linux/windows × amd64/arm64) with `CGO_ENABLED=0` and `-ldflags="-s -w -X main.version=v0.3.0"`
2. Generates `checksums.txt`
3. Creates GitHub release v0.3.0 with `--generate-notes` (commit messages as release notes)

Monitor the release workflow:
```bash

```

Wait for the workflow to complete (typically 2-4 minutes). Then verify:
```bash

```

Expected output: release titled "v0.3.0" with 7 assets listed (6 binaries + checksums.txt).
